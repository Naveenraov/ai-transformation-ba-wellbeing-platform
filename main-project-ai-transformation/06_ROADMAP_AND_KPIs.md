## Roadmap Principles

This roadmap is built on four delivery principles:

**Prove before you scale.** Every automation is piloted, measured, and validated before wider rollout.

**Clinical first.** The highest-impact, highest-risk automations — triage, documentation, referrals — are addressed first because they have the greatest effect on member outcomes.

**Governance parallel, not sequential.** The governance framework is built in Phase 1 alongside the first automation, not after it.

**Report early.** ROI reporting starts in Phase 2. The CEO and CPO see evidence of value before Phase 3 investment decisions are made.

## KPI Framework

### Programme-Level KPIs

| KPI | Baseline (Q1 2026) | Phase 1 Target | Phase 2 Target | Phase 3 Target |
|---|---|---|---|---|
| Triage assessment time (mins) | ~30 mins | 20 mins (–33%) | 15 mins (–50%) | 12 mins (–60%) |
| % routine cases auto-routed | 0% | 0% (design phase) | 40% | 70% |
| Session documentation time (mins) | ~25 mins | 25 mins (baseline) | 10 mins (–60%) | 5 mins (–80%) |
| Referral letter turnaround | 24–48 hrs | 24–48 hrs (baseline) | 4 hrs | Same-day |
| Production prompts in library | 0 | 7 | 15 | 20+ |
| AI governance incidents | 0 | 0 | 0 | 0 |
| Case Manager NPS on AI tools | N/A | Baseline captured | 7+/10 | 8+/10 |

### Financial KPIs

| Metric | Calculation Method | Phase 2 Target | Phase 3 Target |
|---|---|---|---|
| Monthly cost saving — triage | (Baseline mins − actual mins) × FTE rate × volume | €15,000/month | €25,000/month |
| Monthly cost saving — documentation | (Baseline mins − actual mins) × counsellor rate × sessions | €10,000/month | €18,000/month |
| Monthly cost saving — routing | (Baseline routing time − actual) × FTE rate × volume | €8,000/month | €15,000/month |
| AI programme cost (API + tooling) | Anthropic API costs + Salesforce Agentforce licences | Tracked monthly | vs. savings |
| Programme ROI | (Total savings − Total costs) / Total costs × 100 | Positive by Month 6 | 3:1 by Month 12 |

### Clinical Quality KPIs

| KPI | Measurement Method | Target |
|---|---|---|
| Triage accuracy | Agreement between AI recommendation and senior clinician review | 85%+ |
| Safety flag false negative rate | Cases where safety flag should have triggered but didn't | 0% |
| Safety flag false positive rate | Cases where safety flag triggered unnecessarily | <5% |
| Case Manager override rate | % of AI recommendations overridden | Monitored (not minimised — overrides are healthy) |
| Adverse clinical events linked to AI | Incident reports where AI output contributed to poor outcome | 0 |

## Dependency & Risk Summary

| Dependency | Risk if delayed | Mitigation |
|---|---|---|
| DPO Article 9 sign-off | Triage assistant cannot go live | Early DPO engagement — scheduled Week 2 |
| Salesforce Agentforce licences | Case routing agent delayed | Licence procurement in Phase 1 planning |
| IT API integration support | Claude ↔ Salesforce connection delayed | IT resource confirmed in programme kickoff |
| Case Manager availability for UAT | Go-live delayed | UAT scheduled in protected slots; 2-week window |
| Clinical Director sign-off on UAT | Governance gate not met | Clinical Director in steering group from Day 1 |
| Anthropic API stability | AI features unavailable | Graceful degradation designed into all workflows |
