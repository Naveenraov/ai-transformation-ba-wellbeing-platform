## What This Is

A governed, reusable prompt library for Spectrum.Life's two highest-volume operational teams — HR & People Operations and Clinical & EAP Operations.

20 fully documented prompts, each built using the **CRAFT framework** (Context, Role, Action, Format, Tone), tested for consistent output, and governed by six rules that ensure safe, compliant use in a clinical environment.

> A prompt library is not a collection of clever tricks. It is a set of governed, tested, reusable instructions that give every colleague a reliable starting point — so they spend their time reviewing and refining Claude's output, not starting from a blank page every single time.

## The CRAFT Framework

Every prompt is built using CRAFT:

| Letter | Element | Purpose |
|--------|---------|---------|
| **C** | Context | Sets the scene — Spectrum.Life specific, not generic |
| **R** | Role | The expert persona Claude adopts |
| **A** | Action | The specific task — no ambiguity |
| **F** | Format | Consistent, structured, audit-ready output |
| **T** | Tone | Matched to audience — clinical differs from HR differs from executive |

## Governance Rules — Summary

| Rule | Requirement |
|------|-------------|
| Data Handling | Never include identifiable personal data. Case references and initials only. |
| Clinical Boundary | Claude drafts. The qualified clinician decides. Non-negotiable. |
| GDPR Article 9 | Confirm DPA and consent mapping before any clinical data prompt. |
| Output Review | Every output reviewed by a human before action. No exceptions. |
| Prompt Versioning | Changes approved by AI Transformation Partner and department head. |
| Escalation | Unexpected output? Stop. Do not act. Report to AI Transformation Partner. |
