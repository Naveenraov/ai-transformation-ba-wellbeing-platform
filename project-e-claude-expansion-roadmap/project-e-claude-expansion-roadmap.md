## The Expansion Framework — Three Layers

Claude expansion at Spectrum.Life needs to operate across three layers simultaneously. Getting any one layer wrong undermines the others.

```
┌─────────────────────────────────────────────────────────┐
│  LAYER 3 — INTELLIGENCE                                  │
│  Agentforce + Claude API — autonomous multi-step agents  │
│  Connected to Salesforce DHOS — end-to-end workflows     │
├─────────────────────────────────────────────────────────┤
│  LAYER 2 — ADOPTION                                      │
│  Claude Projects — department-specific persistent memory │
│  Claude Skills — governed reusable prompt templates      │
│  Claude Artifacts — document generation at scale         │
├─────────────────────────────────────────────────────────┤
│  LAYER 1 — FOUNDATION                                    │
│  Governance framework — acceptable use, data standards   │
```

**Layer 1 must come first.** Not because it is exciting — it isn't — but because without it, Layer 2 creates inconsistent outputs, ungoverned data handling, and staff who use Claude in ways that create clinical or legal risk. Foundation first, then adoption, then intelligence.

## The Roadmap — 5 Phases Over 12 Months

---

### Phase 1 — Stabilise the Foundation (Months 1–2)

**Objective:** Turn the one live use case into a governed, scalable base. Establish the standards everything else is built on.

**What this is not:** A slow bureaucratic setup phase. Two months is tight. Every week has a deliverable.

#### Key Deliverables

**Claude Governance Framework (Week 1–2)**
A lightweight, practical governance document covering:
- Acceptable use policy — what Claude can and cannot do at Spectrum.Life
- Data handling rules — what can and cannot be passed to Claude (GDPR Article 9 critical here)
- Output review standards — when does Claude output need human review before action?
- Escalation rules — what triggers a governance review?
- Versioning — how are prompts documented, approved, and updated?

This is not a 50-page compliance document. It is a 10-page practical guide every colleague can read and follow.

**Upgrade the HR Teams Chatbot to a Claude Project (Week 2–3)**
The existing Teams chatbot is a single-prompt deployment. Converting it to a full Claude Project gives it:
- Persistent memory of HR policies, procedures, and org structure
- Context about Spectrum.Life's culture, values, and ways of working
- Role-aware responses — the answer for a counsellor differs from the answer for a sales manager
- Consistent tone and format across every interaction

This single upgrade takes the existing chatbot from a FAQ responder to a genuine HR assistant.

**Baseline Measurement (Week 1–4)**
Before expanding, measure the current state:
- How many HR queries does the Teams bot handle per week?
- What is the average time saved per query?
- What percentage of queries are escalated to a human?
- What is staff satisfaction with the current tool?

These are the numbers that prove ROI as the roadmap progresses.

**Prompt Library Version 1 — HR (Week 3–4)**
Document every prompt currently used in the HR chatbot using the CRAFT framework (Context, Role, Action, Format, Tone). This becomes the template for every department that follows.

#### Phase 1 Success Metrics
- Governance framework published and signed off by Lynda and Stephen
- HR Claude Project live and handling 50%+ of HR queries without escalation
- Baseline metrics captured for all active Claude use cases
- Prompt Library V1 — HR complete and under version control

### Phase 2 — First Departmental Expansion (Months 3–5)

**Objective:** Take Claude from one department (HR) to three departments (HR + Clinical Operations + Sales). Prove the model works beyond its starting point.

**Why these three departments first:**

- **Clinical Operations** — highest volume, highest pain, highest strategic value. Case Managers are losing 40-50% of their day to admin. Claude gives it back.
- **Sales & Account Management** — 3,000 clients, majority of revenue growth from UK. Proposal drafting, renewal risk, client reporting — all high-volume, all repetitive.
- **HR** — already live. Phase 2 deepens it, doesn't restart it.

#### Clinical Operations Claude Project

This is the most important deployment in the entire roadmap. The Clinical Operations Claude Project is built around the Case Manager role:

**Persistent context includes:**
- EAP clinical protocols and triage guidelines
- Risk assessment frameworks and escalation thresholds
- GDPR Article 9 data handling rules
- Approved response templates for common presenting issues
- Spectrum.Life's Stepped Care Model — what each pathway means and when to use it

**Capabilities enabled:**
- Inbox triage and classification (see Project D Quick Win 1)
- Case note generation from brief summary input
- Pre-audit readiness review
- In-the-moment reference — "what is the protocol for X?" answered in seconds

**Critical governance note:** This Project is read-only for clinical guidance. Claude assists, summarises, and drafts. It never makes clinical decisions. Every output that informs a clinical action is reviewed by a qualified Case Manager before action is taken.

#### Sales & Account Management Claude Project

**Persistent context includes:**
- Spectrum.Life's service portfolio and pricing tiers
- Client segment profiles — insurer, corporate, university
- Renewal process and commercial terms
- Key client accounts and relationship context
- Salesforce data integration (via API) — account health, usage data, NPS scores

**Capabilities enabled:**
- First-draft proposal generation
- Post-meeting follow-up email drafting
- Client EAP utilisation report generation
- Renewal risk narrative — "here is why this client is at risk and what to say"

#### Prompt Library Expansion — Clinical and Sales (Month 4–5)

Every prompt used in both new Claude Projects is documented in the CRAFT framework and added to the governed Prompt Library. By end of Phase 2, the library covers HR, Clinical, and Sales — the three highest-volume teams in the organisation.

#### Phase 2 Success Metrics
- Clinical Operations Claude Project live — Case Managers saving 60+ minutes/day
- Sales Claude Project live — proposal drafting time reduced by 70%
- Prompt Library covers 3 departments, 30+ governed prompts
- Staff confidence scores improving — measured via monthly pulse survey
- Zero governance incidents — no clinical data mishandled, no GDPR breaches

### Phase 3 — Claude Artifacts and Document Intelligence (Months 5–7)

**Objective:** Activate Claude Artifacts across Finance, Governance, and Marketing. Move Claude from conversational assistant to document intelligence engine.

**Why Artifacts matter at this stage:**

Spectrum.Life produces a significant volume of documents that follow consistent formats but require substantial manual effort: board reports, client utilisation summaries, governance documents, case studies, EAP participation reports. Claude Artifacts turns these from hours of manual work into minutes of review.

#### Finance & Reporting Artifacts

**Weekly revenue narrative** — Claude reads Salesforce financial data and generates a 1-page plain-English performance summary every Monday morning. Stephen opens his week with a clear picture of where the business stands — without anyone spending 3 hours compiling it.

**Automation ROI tracker** — A governed Artifact template that tracks the cost savings and efficiency gains from every automation deployed. This is how Spectrum.Life proves the 40-60% automation target is being hit — with numbers, not claims.

**Board update first drafts** — Claude generates first-draft quarterly board updates from structured financial and operational inputs. Board prep time reduced from days to hours.

#### Clinical Governance Artifacts

**Incident report generator** — Case Manager provides a brief input; Claude generates a structured, audit-ready incident report in the required clinical format. Consistent. Complete. Immediately reviewable.

**Policy update drafts** — When clinical protocols change, Claude generates updated policy documentation from a brief summary of what changed. Governance team reviews and approves.

**Audit preparation summaries** — Before scheduled clinical audits, Claude reviews recent case documentation and generates a pre-audit readiness report. No more surprises.

#### Marketing & Content Artifacts

**Client case study generator** — Sales or account management team provides a structured brief (client, challenge, solution, outcome, key stats). Claude generates a polished first-draft case study. Marketing refines and publishes.

**Thought leadership content** — Claude drafts blog posts, resources, and educational articles from a brief. Editor reviews. Publication cadence increases without headcount increase.

#### Phase 3 Success Metrics
- Finance Artifacts live — board update prep time reduced by 60%+
- Governance Artifacts live — incident report generation time reduced from 45 min to 5 min
- Marketing Artifacts live — content output increased by 50%+ without new headcount
- Automation ROI tracker showing cumulative savings across all deployments

### Phase 4 — Claude Skills and Organisation-Wide Enablement (Months 7–9)

**Objective:** Activate Claude Skills across all departments. Build the internal capability so every colleague can use Claude confidently, safely, and well — without needing to ask Stephen.

**This is the phase where Claude stops being something the AI Transformation Partner does and starts being something every person at Spectrum.Life does.**

#### Claude Skills — What They Are and Why They Matter

A Claude Skill is a reusable, governed prompt template. Instead of every colleague trying to write their own prompts from scratch — getting inconsistent results, using incorrect data handling practices, or simply giving up — Skills give them a tested, approved starting point.

**Examples of Skills built in this phase:**

| Skill Name | Department | What It Does |
|---|---|---|
| EAP Case Note | Clinical | Generates structured case note from brief call summary |
| Renewal Proposal | Sales | Generates first-draft renewal proposal from account data |
| HR Policy Answer | HR | Answers policy questions with source references |
| Incident Report | Governance | Generates structured incident report from brief input |
| Client Report | Account Mgmt | Generates monthly utilisation report from Salesforce data |
| Meeting Summary | All Teams | Generates structured meeting summary from notes |
| Job Description | HR | Generates first-draft JD from role brief |
| Board Update | Finance | Generates board update section from structured data input |
| Case Study Draft | Marketing | Generates case study from structured client brief |
| Wellbeing Resource | Content | Generates educational wellbeing article from topic brief |

Each Skill is documented, version-controlled, approved by the relevant department head, and accessible to every colleague in their Claude Project.

#### AI Enablement Training Programme

Stephen cannot be the only person who understands Claude deeply. This phase builds that capability across the organisation.

**Training structure:**

| Module | Audience | Content | Format |
|---|---|---|---|
| Claude Foundations | All staff | What Claude is, what it can do, what it cannot do, how to use it safely | 30-min async |
| Claude for Your Role | Department-specific | Role-specific use cases, Skills, and prompts | 45-min workshop |

**Target:** Every Spectrum.Life employee completes Claude Foundations and Governance & Data Safety within 60 days of Phase 4 launch.

### Phase 5 — Agentforce Integration and Autonomous Workflows (Months 9–12)

**Objective:** Connect Claude to Salesforce Agentforce. Move from AI-assisted tasks (human reviews every output) to AI-autonomous workflows (Claude acts within defined boundaries without requiring approval for every step).

**This is where the 40-60% automation target becomes achievable.** Phases 1–4 get Spectrum.Life to AI-assisted operations. Phase 5 gets them to AI-autonomous operations — for the tasks where that is safe, appropriate, and commercially valuable.

#### The Three First Agentforce Deployments

**Agent 1 — Clinical Triage Routing Agent**

*What it does:* When a new member contact arrives via any channel, the agent classifies urgency and type, checks Case Manager availability in Salesforce, routes the contact to the appropriate Case Manager, pre-populates the intake form with available member data, and sends a confirmation to the member — all without manual intervention.

*What it does not do:* Make clinical decisions. The routing is administrative. The Case Manager who receives the pre-populated intake form makes all clinical judgements.

*Impact:* Estimated 8-12 minutes saved per intake. At 50+ intakes per day across the team, that is 400-600 minutes of clinical admin time recovered daily.

**Agent 2 — HR Onboarding Agent**

*What it does:* When a new hire is confirmed in Salesforce, the agent triggers a full onboarding workflow: generates the personalised onboarding document pack, sends welcome communications, creates the new hire's Salesforce profile, assigns them to the correct Claude Project for their role, schedules onboarding check-ins with their manager, and tracks completion of required onboarding steps.

*Impact:* HR onboarding admin reduced by 70%. With 100+ new hires coming, this is a significant capacity saving for Lynda's team.

**Agent 3 — Sales Renewal Agent**

*What it does:* 90 days before each contract end date, the agent calculates the renewal risk score, generates a renewal proposal draft, creates a task in Salesforce for the account manager with all context attached, sends an internal alert with the risk summary, and schedules a renewal review meeting.

*Impact:* Renewal management becomes proactive rather than reactive. At-risk accounts identified and actioned 3 months earlier. Renewal rate improvement directly protects revenue.

#### Claude + Agentforce Integration Architecture

```
Member/Staff Contact
        ↓
   Agentforce Agent
   (Salesforce DHOS)
        ↓
   ┌────────────┐
   │   Claude   │  ← Language intelligence layer
   │    API     │    (drafting, classifying, summarising)
   └────────────┘
        ↓
   Salesforce Data
   (CRM, case records,
   utilisation data)
        ↓
   Human Review
   (where required)
        ↓
   Action / Output
```

The architecture is simple by design. Agentforce handles the workflow orchestration. Claude handles the language intelligence. Salesforce holds the data. The human stays in the loop wherever clinical or financial decisions are involved.

- HR onboarding admin reduced by 70%
- Renewal management lead time increased by 90 days per account
- Cumulative automation coverage: 40%+ of identified operational tasks
- Claude usage cost per automated task tracked and optimised

## Investment and Cost Framework

### Claude Usage Cost Model

Claude usage scales with adoption. The cost model below is based on Claude Sonnet (the recommended model for daily operational work) and assumes gradual departmental rollout.

| Phase | Monthly Active Users | Est. Monthly Claude Cost | Value Generated |
|---|---|---|---|
| Phase 1 (Months 1–2) | 10–20 | €200–400 | HR time saving: 15+ hrs/week |
| Phase 2 (Months 3–5) | 30–60 | €600–1,200 | Clinical + Sales time saving: 80+ hrs/week |
| Phase 3 (Months 5–7) | 60–100 | €1,000–2,000 | Document production: 40+ hrs/week saved |
| Phase 4 (Months 7–9) | 150–200 | €2,000–3,500 | Org-wide efficiency: 200+ hrs/week saved |
| Phase 5 (Months 9–12) | 200–300 | €3,000–5,000 | Autonomous workflows: 400+ hrs/week saved |

**ROI principle:** At Spectrum.Life's scale, recovering 400 hours per week of staff time at an average fully-loaded cost of €30/hour represents **€12,000+ per week in recovered productivity** — against a monthly Claude cost of €3,000–5,000 at full adoption. The ROI case is not close.

### What This Does Not Cost

This roadmap does not require:
- New technology platforms (Claude and Salesforce already exist)
- External consultants (all delivery is internal — AI Transformation Partner led)
- Significant capital investment (Claude usage is operational expenditure, not capex)
- Staff redundancy (automation recovers time — it does not eliminate roles)
